Get Axiom up and running on your machine with Docker Compose. The entire stack — frontend, backend, and database — starts with a single command.

## Prerequisites

You need the following tools installed before you begin:

- **Docker** — [Install Docker](https://docs.docker.com/get-docker/)
- **Docker Compose** — included with Docker Desktop; verify with `docker compose version`

<Tip>
  On macOS, you can use [Colima](https://github.com/abiosoft/colima) as a lightweight Docker runtime alternative to Docker Desktop.
</Tip>

## Setup

<Steps>
  <Step title="Clone the repository">
    Clone the Axiom repository and navigate into it:

    ```bash
    git clone https://github.com/invi-bhagyesh/axiom.git
    cd axiom
    ```
  </Step>

  <Step title="Create your .env file">
    Create a `.env` file in the project root. The only variable you need to set manually is your Claude API key for AI flagging. All other values have working defaults.

    ```bash .env
    CLAUDE_API_KEY=sk-ant-...
    ```

    <Note>
      `CLAUDE_API_KEY` is optional. Without it, the AI flagging feature is disabled, but all other annotation functionality works normally.
    </Note>
  </Step>

  <Step title="Build and start the services">
    Build the images and start all three services:

    ```bash
    docker-compose up --build
    ```

    Docker Compose starts MySQL first, waits for it to pass its health check, then starts the backend, and finally the frontend. The first build takes a few minutes to download base images and install dependencies.

    <Tip>
      On subsequent runs, omit `--build` to skip the rebuild step: `docker-compose up`
    </Tip>
  </Step>

  <Step title="Access the application">
    Once all containers are running, open the following URLs in your browser:

    | Service     | URL                          |
    | ----------- | ---------------------------- |
    | Frontend    | http://localhost:3000        |
    | Backend API | http://localhost:8000/docs   |
    | MySQL       | localhost:3306               |

    The interactive API documentation at `http://localhost:8000/docs` is generated automatically by FastAPI and lets you explore and test every endpoint directly from the browser.
  </Step>

  <Step title="Application is ready">
    The database schema and stored procedures are loaded automatically on first start via the MySQL Docker entrypoint. Once all containers are healthy, you can register accounts at `http://localhost:3000/register`.
  </Step>
</Steps>

## First workflow

Log in as the demo requester to explore the platform end to end.

<Steps>
  <Step title="Log in as the requester">
    Go to http://localhost:3000 and sign in with `demo@axiom.dev` / `password123`.

    Axiom redirects you to the requester dashboard, which shows all your projects and their completion status.
  </Step>

  <Step title="Explore the dashboard">
    The dashboard displays each project as a card with a progress ring, task counts, and a flagged-items indicator. You can see projects in **draft**, **active**, and **completed** states.
  </Step>

  <Step title="Open a project">
    Click any project to open the project hub. It has five tabs:

    - **Overview** — status, deadline, and summary stats
    - **Schema** — the annotation fields defined for this project
    - **Data** — uploaded datasets and their items
    - **Annotators** — enrolled annotators, task assignment, and workload view
    - **Results** — inter-annotator agreement scores, AI-flagged items, and the export button
  </Step>

  <Step title="Try annotating">
    Log out and sign in as `ann1@axiom.dev`. The annotator view shows a task queue grouped by project. Open a task to see the dynamic annotation form and submit a response.
  </Step>
</Steps>
